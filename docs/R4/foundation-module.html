2.0 基础组件

基础组件简介了FHIR规范的整体基础架构。要实施FHIR,无论你如何使用，都要用到基础组件中的内容。

基础组件覆盖了FHIR规范中大部分基础文档的内容。 此外，还包括以下资源：

基础框架

  • Resource 资源
  • DomainResource 域资源
  • Basic 基本(最小集)资源
  • Binary 二进制资源
  • Bundle 组合包资源

内容管理资源

  • Questionnaire问卷/问询
  • QuestionnaireResponse问卷答案/问询结果
  • List 列表资源
  • Composition 文档资源
  • DocumentReference (外部)引用文档
  • DocumentManifest 文档集清单

数据交换资源

  • OperationOutcome 操作反馈信息
  • Parameters 操作参数
  • Subscription 订阅
  • MessageHeader 消息头
  • MessageDefinition 消息定义

2.0.1 与其他组件的关系

  • 所有组件都依赖于基础组件
  • 交互组件是通过在基础组件的基础上定义资源的交互方法来建立的。
  • 术语组件 使用代码系统中的概念定义来规定基础格式。
  • 符合性组件通过对基础组件的扩展配置来实现国家和地区的本地化。
  • 安全与隐私组件 向安全和隐私的外部标准提供了链接框架.
  • 实施支持组件 在基础组件之上提供测试和实施参考

2.0.2 研发路线图

基础组件的几个组件现已达到规范状态。在未来18-24个月FHIR第5版的准备期间，将重点关注未达到规范状态的资源，并完成规范性修订， 比如问卷/问询、清单、引用文档和订阅。究竟哪些资源将成为规范发布的候选资源要看FHIR实施者向HL7提供关于实施程序的反馈。
