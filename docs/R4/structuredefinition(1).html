5.3 资源 结构定义 - 内容

工作组:FHIR 基础设施 成熟度: N  规范 (from v4.0.0) 安全类别: 规范 Compartments: 未链接到任何已定义的 compartments

FHIR结构的定义。该资源用于描述基础资源、FHIR中定义的数据类型，也用于描述资源和数据类型的扩展和约束。

5.3.1 范围和用途

结构定义资源描述了一种结构--一组数据元素定义及其相关的使用规则。这些结构定义用于描述FHIR规范本身中定义的内容--参考资料、数据类型、底层基础结构类型，以及如何在实现中使用这些结构。这允许通过结构定义库(相互比较)共享和发布结构的定义，并将其用作代码、报表和UI生成的基础。

注意，作为规范本身的一部分， 全套结构定义 对于所有资源和数据类型，都会发布。

5.3.2边界和关系

  • 结构定义由能力状态实例使用，用于指定如何使用资源
  • 结构定义使用值集来指定编码元素的内容

5.3.3 背景和上下文

在使用此资源之前，实现者应该熟悉FHIR概要中描述的背景和概念。

5.3.3.1 元数据

结构定义资源有一组元数据，这些元数据主要与值集、 能力状态和其他基础结构资源共享。元数据描述结构，并在存储库中注册时帮助查找结构。

url 在规范、模型、设计或实例中引用此结构时，用于标识此结构的标识符。此URL是可以访问结构的位置
identifier 用于标识此结构的其他标识符
version

当在规范、模型、设计或实例中引用此结构版本时，用于标识此结构版本的标识符。这是由结构作者手动管理的任意值，该值应为时间戳。

请注意，可能有多个资源版本的结构具有相同的标识符。由于技术原因，资源将有创建新版本的更新，而声明的版本号需要由作者控制
name 一种计算机就绪名称（如token），用于标识适合代码生成的结构。请注意，此名称（以及与代码生成相关的其他名称，包括元素和片名称、代码等）可能与相关目标语言中的保留字冲突，代码生成器将需要处理此问题
title 标识结构的自由文本自然语言名称
status 结构的状态允许筛选适合使用和不适合使用的结构定义。查看 状态码
experimental 此结构是为测试目的（或 education/evaluation/marketing）而编写的，并非用于真正的用途
date 该结构版本发布的日期
publisher 负责发布结构的个人或组织的详细信息。这有助于建立机构的“权威性/可信性”。
contact 联系方式以帮助用户查找并与发布者通信
description 一种自由文本自然语言对其结构及其使用的描述
purpose 为什么创建这个结构-它的目的是什么
useContext & Jurisdiction 关于用途和使用范围的可计算细节
keyword 一组来自外部术语的术语，可用于帮助对结构定义进行索引和搜索
copyright 使用和/或发布限制
fhirVersion 该结构所基于的FHIR规范的版本。没有必要指定版本，因为大多数结构定义在多个版本中都是有效的，并且可以通过工具很容易地检查结构对特定版本FHIR的有效性。
type 该结构描述的类型。
contextType & context 对于扩展，可以在其中使用扩展的上下文类型。有关详细信息，请参见 定义扩展

注:

  • 结构的name和title不要求全局唯一，但是name和title应该包含一些范围信息(例如AcmeAllergie/Acme Inc.(美国)，过敏列表)
  • 可以为该结构分配多个关键字。它们既可以描述结构，也可以描述结构的焦点，或者两者兼而有之。它们仅仅是通过搜索结构化概念来帮助查找结构。
  • 3种状态代码(draft, active, 和 retired)是与结构使用者相关的代码。作者可能希望使用创作状态扩展以跟踪结构的生命周期

此资源由 DataRequirement, ElementDefinition, Meta, ParameterDefinition, ActivityDefinition, CapabilityStatement, GraphDefinition, ImplementationGuide, MessageDefinition, OperationDefinition, 自身和StructureMap引用

5.3.5 解释说明:

  • 结构表示为元素的平面列表。 element.path 提供 整体结构。
  • 约束中的差异只需要指定它们正在制定规则的元素。其他元素可以根据基本资源中的定义进行推断
  • 在 differential 中指定的元素（以及 snapshot 中的所有元素）必须按如下顺序排列：
    • baseDefinition中的元素出现在派生为“specialization”的StructureDefinition中的新元素之前
    • 元素的顺序必须与baseDefinition相同，子元素的顺序为深度优先。
    • 切片元素的未切片后代出现在切片之前
  • element.short, element.formal, element.comments 和 element.mapping在特定的上下文中为字段提供特定的定义。
  • element.type 用于指定该元素可以使用的类型。如果有 元素提供了多种类型的选择，并且必须有一个以“[x]”结束的名称。 (注意:当用特定的数据类型替换[x]时，第一个字母必须大写。选择类型 总是“骆峰式”。例如:“effectiveDateTime”是正确的，“effectivedateTime”是错误的。) 在对资源进行分析且只指定了一种类型时，元素的名称将更改为include 类型而不是'[x]'。
  • 当使用XPath来定义有关叙述内容之间关系的约束时 以及数据元素的内容，表示规则的元素就是 验证程序将根据其报告错误或警告。
  • 当表达式约束放置在提供类型选择的元素上时，表达式必须使用FHIRPath is operator
  • 对于XPath约束，前缀“f”应用于http://hl7.org/fhir，而“h”表示http://www.w3.org/1999/xhtml". XPath约束是针对XML表示编写的
  • 条件元素用于断言定义在另一个元素上的约束会影响该元素的允许基数。

5.3.5.1 结构定义的不同用途

基本结构定义以多种不同的方式用于支持FHIR规范。StructureDefinition的各种用途由元素kind, type, base 和 url,控制，使用以下基本模式： 

{
  "url": the identity of this structure definition,
  "kind": (primitive-type | complex-type | resource | logical),
  "type": the type being constrained (if it's a constraint)
  "baseDefinition": the structure definition from which this is derived
}

此列表显示了一些示例，并为每个示例提供了指向实际示例的链接：

  1. 数据类型的基本定义(例如： Quantity - XML, JSON):
    {
  "resourceType": "StructureDefinition",
  "url": "http://hl7.org/fhir/StructureDefinition/Quantity",
  "name": "Quantity",
  "kind": "complex-type",
  "baseDefinition": "http://hl7.org/fhir/StructureDefinition/Element"
}
  2. 受约束的数据类型(例如： Money - XML, JSON):
    {
  "resourceType": "StructureDefinition",
  "url": "http://hl7.org/fhir/StructureDefinition/Money",
  "name": "Money",
  "kind": "complex-type",
  "type": "Quantity",
  "baseDefinition": "http://hl7.org/fhir/StructureDefinition/Quantity"
}
  3. 资源的基本定义(例如： Patient - XML, JSON): 
    {
  "resourceType": "StructureDefinition",
  "url": "http://hl7.org/fhir/StructureDefinition/Patient",
  "name": "Patient",
  "kind": "resource",
  "baseDefinition": "http://hl7.org/fhir/StructureDefinition/DomainResource"
}
  4. 对资源的约束(例如： Clinical Document Profile for Composition - XML, JSON): 
    {
  "resourceType": "StructureDefinition",
  "url": "http://hl7.org/fhir/StructureDefinition/clinicaldocument",
  "name": "Clinical Document Profile for Composition",
  "kind": "resource",
  "type": "Composition",
  "baseDefinition": "http://hl7.org/fhir/StructureDefinition/Composition"
}
  5. 基本扩展(标准数据类型)(例如：Extension - XML, JSON): 
    {
  "resourceType": "StructureDefinition",
  "url": "http://hl7.org/fhir/StructureDefinition/Extension",
  "name": "Extension",
  "kind": "datatype",
  "baseDefinition": "http://hl7.org/fhir/StructureDefinition/Element"
}
  6. 定义的扩展(例如：Extension Data Absent Reason - XML, JSON): 
    {
  "resourceType": "StructureDefinition",
  "url": "http://hl7.org/fhir/StructureDefinition/data-absent-reason",
  "name": "Data Absent Reason",
  "kind": "complex-type",
  "type": "Extension",
  "baseDefinition": "http://hl7.org/fhir/StructureDefinition/Extension"
}
  7. 对定义的扩展的约束(目前没有定义的示例)：
    {
  "resourceType": "StructureDefinition",
  "url": "http://example.org/fhir/StructureDefinition/race",
  "name": "Race codes used by institution (a subset of meaningful use codes)",
  "kind": "complex-type",
  "type": "Extension",
  "baseDefinition": "http://hl7.org/fhir/us/core/StructureDefinition/us-core-race"
}

在这个列表中，类型1、3和5的结构定义只能由FHIR规范本身定义。其他类型的结构定义是(或可能)由规范创建的，但也可以由其他实现者定义。

5.3.5.2 约束类型规则

当结构是约束时(derivation = constraint), 见 扩展和限制资源 适用的规则。

5.3.5.4 逻辑模型

结构定义用于定义FHIR的基本结构：数据类型、资源、扩展和概要文件。相同的定义结构也可以用于定义任何具有类型化节点的有向无圈图的任意结构，其中原语类型是fhir规范定义的结构。

此技术有多种用途:

  • 描述所有的专用内容模型
  • 用FHIR的方式描述现存HL7内容模型 (如：v2, CDA)
  • 描述在FHIR中使用的通用设计模式
  • 定义内容模型以支持映射语言
